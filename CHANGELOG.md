# CHANGELOG

## 2026-03-22 - 个人网站现代化重构

### 改了什么
- 重构信息架构：从单页 Markdown 阅读器升级为 `首页 + 博客列表 + 文章详情页`。
- 新增 `posts/index.json`，统一管理文章元数据（标题、日期、标签、摘要、阅读时长、是否精选）。
- 首页新增 Hero、About、Projects、Skills、Contact 等模块，强化个人定位与内容入口。
- 新增 `blog.html`，支持按标签筛选文章。
- 每篇文章新增独立阅读页 `posts/*.html`，支持目录、阅读进度、代码复制、返回顶部。
- 统一重写样式系统（明暗主题、卡片体系、响应式、中文阅读优化、可访问性基础）。
- 新增 SEO 与站点基础文件：`robots.txt`、`sitemap.xml`、`favicon`、OG/meta。
- GitHub Pages 工作流从 Jekyll 构建改为静态文件直接部署，简化部署链路。
- 重写 README，补充本地运行、更新文章、部署说明。

### 为什么这样改
- 降低维护成本：新增内容不再写死在 JS 数组，改为 `Markdown + index.json`。
- 提升阅读与展示质量：首页不再像文档容器，结构更清晰，内容入口更明确。
- 保证长期可持续：保持纯静态方案，不引入重型框架和后端。
- 保持部署简单：继续兼容 GitHub Pages，一次推送即可自动发布。

### 后续可扩展
1. 为文章增加 front matter 自动解析（减少手动维护 `posts/index.json`）。
2. 增加文章全文搜索（可用轻量 JS 索引）。
3. 增加项目数据文件（与文章同样走 JSON 驱动）。
4. 加入轻量统计（例如 Plausible/Umami）替代本地 PV 计数。
